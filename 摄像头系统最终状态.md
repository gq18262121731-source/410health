## 🔧 核心问题与解决方案

### 问题1：OpenCV异步冲突 ✅ 已解决
**症状**：主后端无法直接访问本地摄像头
```
[ WARN] global cap.cpp:459 cv::VideoCapture::open VIDEOIO(DSHOW): raised unknown C++ exception!
```

**解决方案**：
- 创建独立摄像头服务进程 (`camera_service_standalone.py`)
- 运行在端口8001
- 主后端通过HTTP获取快照

**文件**：
- `camera_service_standalone.py` - 新增
- `backend/config.py` - 添加 `camera_local_http_url` 配置
- `backend/services/camera_service.py` - 修改 `capture_local_jpeg()`

### 问题2：WebSocket端口错误 ✅ 已解决
**症状**：前端连接到错误的WebSocket端口
```
WebSocket connection to 'ws://localhost:9000/ws/camera' failed
```

**根本原因**：
- 前端运行在9000端口
- 后端运行在8000端口
- 前端自动推导WebSocket地址为 `ws://localhost:9000`
- 但WebSocket服务器在 `ws://localhost:8000`

**解决方案**：
创建前端环境配置文件：
```env
# frontend/vue-dashboard/.env
VITE_API_BASE=http://localhost:8000/api/v1
VITE_WS_BASE=ws://localhost:8000
```

**文件**：
- `frontend/vue-dashboard/.env` - 新增

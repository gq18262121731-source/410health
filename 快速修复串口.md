# 串口占用问题 - 快速修复

## 问题原因

**串口被占用：`PermissionError(13, '拒绝访问')`**

COM3和COM4被其他程序占用，后端无法连接串口采集数据。

## 解决方法

### 方法1：运行修复脚本（最快）

```bash
scripts\fix_serial_port.bat
```

脚本会：
1. 停止所有Python进程
2. 等待串口释放
3. 重启后端服务

### 方法2：手动修复

#### 步骤1：停止所有Python进程

```powershell
taskkill /F /IM python.exe
```

#### 步骤2：等待3秒让串口释放

```powershell
timeout /t 3
```

#### 步骤3：重启后端

```bash
python run.py
```

### 方法3：只重启后端（如果方法1/2不行）

1. 按 `Ctrl+C` 停止当前后端
2. 关闭所有Python窗口
3. 重新运行：`python run.py`

## 如果还是不行

### 检查1：确认串口号

```powershell
# 查看可用串口
Get-WmiObject Win32_SerialPort | Select-Object DeviceID,Description
```

### 检查2：关闭串口调试工具

可能占用串口的程序：
- Arduino IDE
- PuTTY
- Tera Term
- 串口调试助手
- 其他Python脚本

### 检查3：重启电脑

如果串口一直被占用，重启电脑可以彻底释放。

## 预防措施

1. **只运行一个后端实例**
2. **使用前关闭串口调试工具**
3. **后端崩溃后等待几秒再重启**

## 常见问题

### Q: 为什么会串口被占用？

A: 可能原因：
1. 之前的后端进程没有正常退出
2. 同时运行了多个后端实例
3. 串口调试工具在使用
4. 系统没有及时释放串口

### Q: 如何避免串口占用？

A: 
1. 停止后端时使用 `Ctrl+C` 而不是直接关闭窗口
2. 确认进程完全退出后再重启
3. 不要同时运行多个后端

### Q: 数据模式配置正确吗？

A: 检查 `.env` 文件：
```env
DATA_MODE=serial
SERIAL_ENABLED=true
USE_MOCK_DATA=false
SERIAL_PORT=COM3
```

## 技术细节

### 串口占用检测

Windows系统中，串口是独占资源：
- 一次只能被一个程序打开
- 程序崩溃可能不释放串口
- 需要等待系统超时（通常几秒）

### 双串口模式

当前配置使用双串口：
- `COM3`: 响应模式数据
- `COM4`: 广播模式数据

两个串口都必须可用。

### 串口释放时间

- 正常退出：立即释放
- 异常退出：3-5秒后释放
- 系统卡死：可能需要重启

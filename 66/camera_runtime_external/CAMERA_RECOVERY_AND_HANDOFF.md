# xstrive / XSWCAM-WB4MP 恢复与交接说明

## 当前已经确认的事实

1. 这台摄像头来自 xstrive / 迅思维体系。
2. 你新增的说明书资料明确写了：
   - 供电：`DC12V 2A`
   - 协议：`RTSP / ONVIF / TCP/IP / DHCP / RTMP`
   - 有线网络：`1 个 RJ45 10M/100M 以太网口`
3. 你电脑当前的网卡状态里，`以太网` 仍然是 `Disconnected`。
4. `设备搜索助手v3.0` 的日志也说明它只在 Wi-Fi 网卡 `10.12.39.151` 上空转，没有发现网线侧设备。

这意味着：

```text
当前最大问题不是密码，而是摄像头没有和电脑形成有效物理链路。
```

在这个状态下，无法从电脑侧读取摄像头配置，也无法验证 RTSP、ONVIF 或密码。

## 关于“密码”这件事，能做和不能做的边界

能做的：

1. 用厂家 App 正常绑定后设置一个新的“明文密码”。
2. 用默认账号 `admin` 和少量常见默认密码做有限验证。
3. 如果无法绑定，按厂商说明恢复出厂，然后重新绑定、重新设置密码。
4. 用电脑侧验证新的 RTSP/ONVIF 是否可用。

不能做的：

1. 不能从 SN 反推出密码。
2. 不能从电脑“破解”现有密码。
3. 不能在链路未建立时假装已读到设备配置。

## 现在最可能成功的恢复路径

### 路线 A：最推荐

```text
摄像头接 12V 电源
摄像头网线接路由器 LAN
电脑接同一个路由器
手机 App 添加设备
开启/设置明文密码
电脑用 RTSP / ONVIF 验证
```

为什么推荐：

1. 路由器会自动给摄像头分配 IP。
2. `设备搜索助手` 更容易发现设备。
3. 手机 App 的绑定成功率最高。
4. 后续电脑调试也最稳定。

### 路线 B：如果你只能直连电脑

```text
摄像头接 12V 电源
摄像头网线直连电脑
电脑手动设置静态 IP
扫描常见网段和端口
如果找到了设备，再验证默认凭据或恢复出厂
```

注意：

1. 直连电脑不代表电脑网口能给摄像头供电。
2. 普通电脑网口通常不能替代 PoE。
3. 如果没有独立电源，直连永远不会成功。

## 从快启 PDF 提取到的关键恢复信息

本地 PDF：`tools/VStarcam_C_series_quick_start.pdf`

从文档提取出的重点：

1. App 名称是 `Eye4`。
2. 第一次使用建议手机连接 `2.4GHz Wi-Fi`。
3. 如果听不到类似 “Please use APP for WiFi configuration”，说明设备状态不对，需要复位。
4. 复位方法：

```text
找到 Reset 按钮
用针按住约 5 秒
听到 "Reset completed"
```

5. App 无法添加时，文档建议改用网线方式尝试。

## 推荐先验证的默认信息

这些不是“破解”，而是厂商/资料里出现过的少量常见默认组合，可用于恢复后第一次验证：

```text
账号：admin
候选密码：888888
RTSP 端口：10554
ONVIF 端口：10080
主码流：/tcp/av0_0
子码流：/tcp/av0_1
```

一旦你在 App 里开启了“明文密码”，以后以你自己设置的新密码为准，不再优先尝试默认密码。

## 你自己现在该做的事

### 第一步：先解决物理链路

目标：

```text
把 Windows 里的“以太网”从 Disconnected 变成 Up
```

如果做不到，后续一切停止。

### 第二步：优先改成“接路由器 + 手机 App”

如果现场允许，直接这么接：

```text
摄像头 -> 12V 电源
摄像头 -> 路由器 LAN
电脑 -> 同一路由器
手机 -> 同一路由器 Wi-Fi
```

### 第三步：必要时恢复出厂

如果 App 一直加不上，执行：

1. 上电 30 秒
2. 用针按住 Reset 约 5 秒
3. 听到 `Reset completed`
4. 重新用 Eye4 添加

### 第四步：设置新的明文密码

一旦 App 成功绑定，立刻去设置里找：

```text
明文密码
本地密码
RTSP 密码
ONVIF
网络访问
```

然后设置你自己的新密码。

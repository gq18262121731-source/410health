# 告警分诊与优先级处理手册

适用场景：多源告警汇总、社区值守、家属通知、交班。

## 分诊原则

- 同时看事件危险度、持续时间、证据强度和对象脆弱性。
- 不要只按单一告警类型排序，要看多信号叠加。
- 有生命危险嫌疑时，优先级应高于设备故障和数据缺口。

## 一级优先处理

典型情形：

- SOS。
- 跌倒后长时间不起。
- 低血氧严重下降并伴呼吸困难或意识异常。
- 高热伴意识差、抽搐、持续胸痛、卒中疑似表现。

建议动作：

- 立即联系现场人员和家属。
- 需要时直接升级急救。
- 尽量保留多模态证据，包括时间点、视频、最近生命体征和告警记录。

## 二级优先处理

典型情形：

- 中高风险生命体征异常连续出现。
- 姿态异常持续存在但未完全确认跌倒。
- 同一设备短时间多次告警。
- 高龄独居对象的离线或长时间无数据。

建议动作：

- 优先电话确认或安排人工复核。
- 跟进告警是否自行恢复。
- 记录是否需要转为一级事件。

## 三级优先处理

典型情形：

- 单次轻度异常。
- 已恢复且有合理解释的短时事件。
- 已知误报场景。

建议动作：

- 保留记录。
- 继续观察趋势。
- 若重复出现，再提升优先级。

## 数据质量类单独标记

以下情况应单独标成“链路/数据风险”，不要混写成健康异常：

- 设备离线。
- 电量过低。
- 数据长时间中断。
- 摄像头黑屏或卡顿。
- 传感器佩戴松动或疑似测量噪声。

## 交班最少要写清楚的内容

- 当前最高优先级对象是谁。
- 为什么优先。
- 已做了什么。
- 还差什么动作。
- 下一次复核时间或负责人。

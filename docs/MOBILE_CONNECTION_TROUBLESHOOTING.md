# 手机端连接问题排查指南

## 问题症状

手机端无法连接到服务器

## 可能的原因和解决方案

### 1. 检查后端是否正在运行

**检查方法**：
```bash
# 在电脑上访问
http://localhost:8000/healthz
```

**预期结果**：
```json
{"status":"ok","app":"AIoT智慧康养健康监测系统"}
```

**如果失败**：
- 启动后端：`python run.py`
- 确认端口8000没有被占用

### 2. 检查防火墙设置 ⚠️ **最常见原因**

Windows防火墙可能阻止了外部设备访问。

**解决方法**：

#### 方法A：临时关闭防火墙（测试用）
1. 打开"Windows安全中心"
2. 点击"防火墙和网络保护"
3. 点击当前活动的网络（专用网络或公用网络）
4. 关闭"Microsoft Defender 防火墙"
5. 测试手机是否能连接

#### 方法B：添加防火墙规则（推荐）
```powershell
# 以管理员身份运行PowerShell，执行：
New-NetFirewallRule -DisplayName "AIoT Health Backend" -Direction Inbound -LocalPort 8000 -Protocol TCP -Action Allow
```

或者手动添加：
1. 打开"Windows Defender 防火墙"
2. 点击"高级设置"
3. 点击"入站规则" → "新建规则"
4. 选择"端口" → 下一步
5. 选择"TCP"，输入端口"8000" → 下一步
6. 选择"允许连接" → 下一步
7. 全选（域、专用、公用）→ 下一步
8. 名称填"AIoT Health Backend" → 完成

### 3. 确认电脑和手机在同一局域网

**检查方法**：

#### 在电脑上查看IP地址
```bash
# Windows
ipconfig

# 查找"无线局域网适配器 WLAN"或"以太网适配器"
# 记录IPv4地址，例如：192.168.8.253
```

#### 在手机上检查
1. 打开手机WiFi设置
2. 查看连接的WiFi名称
3. 确认与电脑连接的是**同一个WiFi**

**常见问题**：
- ❌ 电脑连有线，手机连WiFi（可能在不同网段）
- ❌ 手机连的是5G网络而不是WiFi
- ❌ 电脑和手机连的是不同的WiFi

### 4. 配置手机APP的服务器地址

#### 获取服务器IP
运行后端时会显示：
```
Mobile App should connect to Server IP:
  http://192.168.8.253:8000
```

#### 在手机APP中配置
1. 打开APP
2. 在登录页面点击**齿轮图标**（右上角）
3. 进入"服务器设置"
4. 输入：
   - **主机**：`192.168.8.253`（你的电脑IP）
   - **端口**：`8000`
   - **协议**：`http`
5. 点击"测试连接"
6. 如果成功，点击"保存"

### 5. 测试连接

#### 在手机浏览器中测试
在手机浏览器中访问：
```
http://192.168.8.253:8000/healthz
```

**如果能访问**：
- ✓ 网络连接正常
- ✓ 防火墙配置正确
- → 问题可能在APP配置

**如果不能访问**：
- ✗ 检查防火墙（最常见）
- ✗ 检查是否在同一局域网
- ✗ 检查后端是否运行

### 6. 检查CORS配置

CORS配置已经设置为允许所有来源：

```python
# backend/main.py
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 允许所有来源
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
    expose_headers=["*"],
)
```

如果仍有CORS问题，重启后端服务。

### 7. 检查后端监听地址

确认后端监听在所有网络接口：

```bash
# .env文件中
HOST=0.0.0.0  # ✓ 正确：监听所有接口
# HOST=127.0.0.1  # ✗ 错误：只监听本地
```

```bash
# run.py启动命令
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

## 快速诊断流程

### 步骤1：在电脑上测试
```bash
# 访问
http://localhost:8000/healthz
```
- ✓ 成功 → 进入步骤2
- ✗ 失败 → 启动后端服务

### 步骤2：在手机浏览器测试
```bash
# 访问（替换为你的电脑IP）
http://192.168.8.253:8000/healthz
```
- ✓ 成功 → 进入步骤3
- ✗ 失败 → 检查防火墙和网络

### 步骤3：在APP中测试连接
1. 打开APP
2. 点击齿轮图标
3. 输入服务器地址
4. 点击"测试连接"
- ✓ 成功 → 保存并登录
- ✗ 失败 → 检查APP配置

## 常见错误信息

### "连接超时"
**原因**：
- 防火墙阻止
- 不在同一局域网
- 后端未启动

**解决**：
1. 检查防火墙
2. 确认同一WiFi
3. 确认后端运行

### "连接失败"
**原因**：
- IP地址错误
- 端口错误
- 后端未启动

**解决**：
1. 确认IP地址正确
2. 确认端口是8000
3. 重启后端服务

### "服务器有响应，但这不是可用的后端健康检查接口"
**原因**：
- 输入的地址不是后端服务地址
- 端口错误

**解决**：
1. 确认地址格式：`http://IP:8000`
2. 不要加`/api/v1`等路径

## 网络配置示例

### 正确的配置
```
电脑IP：192.168.8.253
手机IP：192.168.8.xxx（同一网段）
WiFi：同一个
后端：http://192.168.8.253:8000
```

### 错误的配置
```
❌ 电脑：192.168.1.100
   手机：192.168.8.200
   （不同网段）

❌ 电脑：有线网络
   手机：WiFi
   （可能不同网段）

❌ 后端：http://localhost:8000
   （手机无法访问localhost）
```

## 调试命令

### 查看电脑IP
```bash
# Windows
ipconfig

# 查找"IPv4 地址"
```

### 测试端口是否开放
```bash
# 在电脑上
netstat -ano | findstr :8000

# 应该看到：
# TCP    0.0.0.0:8000    0.0.0.0:0    LISTENING
```

### 测试防火墙
```bash
# 在手机上用浏览器访问
http://电脑IP:8000/healthz

# 如果能访问，说明防火墙配置正确
```

## 总结

**最常见的3个问题**：
1. 🔥 **防火墙阻止**（80%的情况）
2. 📡 **不在同一WiFi**（15%的情况）
3. ⚙️ **APP配置错误**（5%的情况）

**解决顺序**：
1. 先检查防火墙
2. 再检查网络
3. 最后检查APP配置

**快速测试**：
在手机浏览器访问 `http://电脑IP:8000/healthz`
- 能访问 → APP配置问题
- 不能访问 → 防火墙或网络问题

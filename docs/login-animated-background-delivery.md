# UI-008 + UI-009 Delivery Notes

本文件记录本轮登录链路和动态背景的实际落地方式，作为前端继续扩展时的直接参考。

## 1. 登录链路结构

登录页主容器：
- `frontend/vue-dashboard/src/App.vue`

注册链路组件：
- `frontend/vue-dashboard/src/components/PublicRegistrationPanel.vue`

登录卡片固定为 5 段：
1. 标题区
2. 身份选择区
3. 输入区
4. 主按钮区
5. 辅助链接区
6. 注册入口区

说明：
- 虽然代码里额外保留了“演示账号快捷进入”小面板，但它放在主卡片外部，不干扰 5 段主结构。
- 登录卡片使用轻玻璃拟态，输入框区和功能区再用更浅的内层卡片包裹。

## 2. 注册链路结构

注册流程固定为 4 步：
1. 身份选择
2. 注册账号
3. 资料完善
4. 完成态

实现原则：
- Step 2 只聚焦账号信息，不把资料完善塞进同一步。
- Step 3 承接角色相关资料：
  - 老人：房间号、年龄、绑定计划
  - 家属：关系类型、首屏关注、绑定计划
  - 社区：值守班次、值守站点、首屏偏好
- Step 4 必须保留，不允许注册成功后直接自动关闭。
- 回填登录发生在 Step 4 的主按钮点击时，而不是 API 成功的瞬间。

## 3. 动态背景分层

背景分三层：
1. 静态底图：`frontend/vue-dashboard/src/assets/bg.jpg`
2. 背景柔光覆盖层：`demo-theme.css` 中的 `.login-background-overlay`
3. SVG 心电图层：在 `App.vue` 内联 SVG，并用沿线高亮完成扫光

实现细节：
- 主 ECG 线：`login-ecg-layer--primary`
- ECG 高亮扫光动画：`ecgDashTravel`

当前策略：
- 默认不启用粒子
- 只保留一条主 ECG 线
- 渐变扫光只沿 ECG 线移动，不做额外全屏扫光
- 避免出现满屏粒子、长拖尾、游戏感特效

## 4. 响应式与降级

桌面端：
- 保留完整底图
- 保留单主 ECG 与沿线扫光
- 登录卡片保持中心焦点

平板端：
- 保留底图
- 保留主 ECG 和更轻的沿线扫光

手机端：
- `<= 720px`：进一步降低 ECG 存在感
- `<= 560px`：直接隐藏 ECG 动效，只保留静态底图与卡片

无障碍 / 性能降级：
- `prefers-reduced-motion: reduce` 下关闭 ECG 高亮扫光动画

## 5. 视觉基线

基础样式文件：
- `frontend/vue-dashboard/src/demo-theme.css`

本轮新增重点：
- `--login-backdrop-image`
- 轻玻璃拟态登录卡片
- 登录链路专用背景层、扫光层、ECG 动效层
- 登录辅助链接、帮助面板、注册入口区样式

## 6. 后续扩展建议

如果后续补入用户真实参考图：
- 优先替换 `bg.jpg`
- 尽量不改 ECG 和卡片结构
- 先看卡片对比度是否仍足够，再决定是否微调扫光和线条透明度

如果后续要加粒子：
- 桌面端同时可见不超过 8
- 移动端同时可见不超过 3
- 只允许沿 ECG 附近轻微漂移或短暂发光

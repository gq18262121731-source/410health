# 智能体工程师协作文档（中文）

Last updated: 2026-03-22

## 角色定位

你是 `AG-` 角色，也就是智能体工程师。

你的职责不是推进任意前端页面或健康模型细节本身，而是负责把“智能体相关能力”收口成团队可稳定协作的公共能力，重点包括：

- 智能体公共协议
- 输出清洗与归一化
- 前后端对齐
- 报告链与对话链的返回结构
- 智能体相关测试护栏

## 当前任务

当前任务板中没有新的 `OPEN` 状态 `AG-*` 任务。

当前默认状态是：

- 待命
- 进行协议兼容性审查
- 进行前端消费方式审查
- 进行报告链与对话链的结构护栏审查

当前已经落地、需要持续守住的能力有两条：

1. 普通智能体对话链
   - 公共返回结构：`answer / analysis / references`
2. 正式时间段健康报告链
   - 公共接口：`POST /api/v1/chat/report/device`
   - 公共报告结构：稳定、可供前端直接消费

## 当前不要做什么

在没有新的正式派发前，不要主动做下面这些事：

- 不要主动扩充 AG 公共返回字段
- 不要主动修改前端页面视觉
- 不要主动替 FE 决定报告页长什么样
- 不要主动替 HM 设计新的健康特征字段
- 不要主动替 OM 重写模型路由策略
- 不要把正式健康报告重新并回普通 `/chat/analyze`
- 不要把 prompt、tool、orchestration、internal fields 暴露给前端
- 不要在没有任务板支持的情况下自行开启下一轮实现

## 你依赖谁

### 直接依赖

- `OM-*`
  - 决定普通对话模型与正式报告模型的内部路由策略
  - 决定 prompt package 与本地 RAG 方向
- `HM-*`
  - 提供正式报告所需的规范化健康模型上下文
  - 提供评分、异常解释、持续异常状态等内部依据
- `BE-*`
  - 提供后端 API 路由、配置、鉴权、持久化和契约落地
- `FE-*`
  - 提供真实页面消费方式，暴露兼容性问题
- `TE-*`
  - 提供协议和回归测试结果，帮助确认 AG 收口是否真实成立

### 当前判断依赖状态

- 目前不依赖别人先完成才能维持现有 AG 能力
- 一旦进入“普通对话模型 / 正式报告模型正式拆路由”的下一轮，就会直接依赖：
  - `OM-010`

## 谁依赖你

### 前端工程师依赖你

- 获取稳定的智能体公共响应结构
- 获取稳定的报告响应结构
- 确认哪些字段可直接消费，哪些字段不能依赖

### 测试工程师依赖你

- 获取可回归的协议定义
- 获取清洗规则边界
- 获取正式报告结构断言目标

### 离线模型工程师依赖你

- 在模型路由变化时由你保证对外协议不漂移

### 健康模型工程师依赖你

- 在健康模型内部上下文变化时由你保证前端不直接吃内部字段

## 当前工作边界

### 你负责

- `agent/*` 中与协议、清洗、报告组装直接相关的部分
- 智能体返回结构的最小公共契约
- 报告结构的稳定化
- 智能体与前端消费方式之间的兼容性确认
- 智能体相关测试对齐

### 你不负责

- 登录/注册 UI 拆页本身
- 全站视觉系统
- 健康评分与异常模型本体
- 本地模型选型与路由策略本体
- BLE 网关接入实现

## 正式健康报告固定规则

正式健康报告当前必须遵守：

- 产品展示上与普通 AI 对话分开
- 公共入口固定走：`POST /api/v1/chat/report/device`
- 公共结构必须稳定
- 前端应该直接消费结构化字段，不做 prompt 级推断

正式报告对前端暴露的结构应稳定包含：

- `report_type`
- `device_mac`
- `subject_name`
- `device_name`
- `generated_at`
- `period`
- `summary`
- `risk_level`
- `risk_flags`
- `key_findings`
- `recommendations`
- `metrics`
- `references`

正式报告绝对不能暴露：

- prompt text
- tool traces
- orchestration metadata
- internal model fields
- reasoning text

## 兼容性审查触发条件

出现以下任一情况时，AG 必须介入：

1. FE 修改前端消费方式，导致现有响应结构不再优雅可用
2. OM 修改模型路由或 prompt package，导致返回文本风格或清洗边界变化
3. HM 修改内部报告上下文，导致报告结构需要新增或重排字段
4. BE 修改报告接口、响应模型或鉴权策略
5. TE 回归测试显示：
   - prompt 泄露
   - tool/internal field 泄露
   - 报告结构漂移
   - 普通对话与正式报告重新混用

5. 你依赖谁 / 谁依赖你
6. 当前是否已准备好接收下一轮正式任务

# 跌倒检测 ROI 与现场样本校准指南

日期：2026-04-30

## 目标

ROI（Region of Interest，检测关注区域）用于约束“哪些模型事件可以进入系统告警”。它不修改模型推理流程，不降低模型原始检测能力；模型仍然会完整分析摄像头画面并写入事件日志，系统只是在生成平台告警前，根据现场活动区域过滤明显不合理的边缘误报。

当前默认启用配置：

```env
FALL_DETECTION_ROI_ENABLED=true
FALL_DETECTION_ROI_RECT=0.18,0.32,0.82,0.94
FALL_DETECTION_ROI_MIN_OVERLAP=0.50
FALL_DETECTION_FRAME_WIDTH=2304
FALL_DETECTION_FRAME_HEIGHT=1296
FALL_DETECTION_MIN_ALERT_SCORE=0.0
```

`FALL_DETECTION_ROI_RECT` 使用归一化坐标：

```text
x1,y1,x2,y2
```

其中左上角是 `0,0`，右下角是 `1,1`。例如 `0.18,0.32,0.82,0.94` 表示只把画面中间偏下的主要活动区作为告警有效区域。

## 为什么这样做

近期现场事件中，误报框大量出现在画面左边缘、右边缘、底边或强光窗帘附近。这类区域通常不是老人主要活动区域，却容易因为遮挡、曝光、边界运动触发“疑似跌倒”。ROI 可以把这些区域挡在系统告警之外，同时保留模型日志供复盘。

## 校准流程

1. 固定摄像头角度

   摄像头位置、俯仰角、焦距和画面方向确定后再校准。调整摄像头后必须重新校准 ROI。

2. 采集正常现场样本

   至少采集 10 到 20 分钟：

   - 无人在画面内
   - 老人正常走路
   - 坐下、起身
   - 弯腰拾物
   - 靠近窗帘、床边、桌边等容易误判的位置

3. 采集安全模拟跌倒样本

   在有人保护的条件下采集：

   - 主要活动区内缓慢倒地
   - 床边或沙发边滑落
   - 跌倒后短暂停留
   - 跌倒后自行起身

4. 复盘截图和事件

   查看：

   ```text
   data/fall_events/camera_events.jsonl
   data/fall_events/snapshots/
   ```

   记录每个误报或漏报的 bbox 位置，判断是否属于有效老人活动区域。

5. 调整 ROI

   调整原则：

   - 尽量覆盖老人真实活动范围。
   - 排除天花板、灯具、窗帘强光、固定墙角、画面极边缘。
   - 床、沙发、卫生间门口等高风险区域必须保留。
   - 不要为了压低误报把 ROI 缩得过小。

6. 重启后端并验证

   修改 `.env` 后重启后端，让配置生效。打开：

   ```text
   http://127.0.0.1:8000/api/v1/camera/fall-detection/status
   ```

   确认 `roi.enabled=true`，`roi.rect` 与 `.env` 一致。

## 验收标准

| 场景 | 预期结果 |
| --- | --- |
| 画面无人 10 分钟 | 不弹出跌倒告警 |
| 正常走路、坐下、弯腰 | 不弹出跌倒告警，或只产生低优先级模型日志 |
| ROI 内模拟跌倒 | 系统产生跌倒告警、截图和事件 |
| ROI 外窗帘/强光/边缘扰动 | 允许模型记录事件，但系统不弹出告警 |
| 点击“确认处理并解除警报” | 当前活跃跌倒告警消失，不立即重复弹出同一批旧告警 |

## 后续优化建议

1. 在前端监控画面叠加 ROI 边框，让校准更直观。
2. 增加“校准模式”，从最近 50 条事件自动统计误报热点。
3. 为每个摄像头保存独立 ROI，支持多摄像头部署。
4. 用现场样本建立回放测试集，每次模型或阈值调整后自动跑一遍。

## 2. 当前任务

当前设备注册工程师处于“准备态 / 待命态”。

当前主判断：

- 本轮专项主战场是前端统一风格、页面结构收口、登录/注册链条拆分。
- 设备注册线不是当前主资源投入方向。
- 在用户没有明确要求“真实蓝牙网关演示”或“网关接收链路正式落地”前，设备注册工程师不主动抢主线实现资源。

当前设备注册侧任务状态：

- 当前没有开放中的 `DR-*` 编码主任务。
- 已经完成并需要持续维护口径的内容：
  - 设备注册 / 绑定 / 解绑 / 换绑 / 删除规则
  - 多设备规则：
    - 一位老人可以挂多台设备
    - 但同型号设备只能挂一台
  - 未注册设备通过网关上报时的 quarantine 策略
  - BLE 网关 HTTP 接入前置准备文档

## 3. 当前不要做什么

在没有新一轮正式派发前，设备注册工程师当前不要主动做这些事：

- 不抢前端统一风格与页面拆分主资源。
- 不主动进入登录页、注册页、图表风格、页面视觉统一的主实现。
- 不主动实现真实 BLE HTTP 接收端。
- 不主动拍板后端公共 HTTP / MessagePack 接口字段细节。
- 不主动做真实硬件接入测试。
- 不因为 PowerShell 中文乱码就判断源码损坏。
- 不跨角色接手 `FE-*`、`UI-*`、`BE-*`、`TE-*` 的实现，除非先上报并获得继续推进。

专项期特别规则：

- 如果涉及跨角色继续做实现，必须先上报，再动手。

## 4. 当前职责边界

设备注册工程师的主职责是：

- 设备注册规则
- 设备绑定、解绑、换绑、删除规则
- 设备生命周期一致性
- 设备台账与绑定审计规则
- 设备与老人的基数规则
- 网关接入前置准备中的设备注册侧策略
- 未注册设备的 quarantine / pending-registration 规则

设备注册工程师不直接拍板但需要对齐的内容：

- 后端 HTTP / MessagePack 正式公共契约
- 网关请求级元数据落库结构
- 真正的 HTTP receiver 端点实现
- 真实硬件联调计划与执行

## 5. 当前已确认规则

### 5.1 权限规则

- 未登录用户不能做设备管理写操作。
- `family`、`community`、`admin` 可以做设备管理写操作。
- `elder` 不能做设备管理写操作。

### 5.2 设备规则

- 一位老人可以绑定多台设备。
- 但同型号设备只能绑定一台。
- 删除设备后：
  - 设备记录删除
  - 绑定历史删除
  - 后续如果同一 MAC 再上报，按“未注册设备”处理
  - 若要继续绑定，必须重新注册再绑定

### 5.3 网关准备规则

- 当前 BLE HTTP 网关接入还未正式实现。
- 设备注册侧已经完成准备态设计，见：
  - `docs/ble-gateway-http-prep.md`
- 正式 HTTP 契约文档见：
  - `docs/ble-gateway-http-contract.md`

### 5.4 未注册设备策略

formal mode 下：

- 不自动注册
- 不进入正常健康数据流
- 不触发告警
- 不更新设备在线状态
- 进入 quarantine / pending-registration 证据落点

## 6. 你依赖谁

设备注册工程师当前主要依赖：

- 后端工程师
  - 当需要把设备注册侧规则落成正式 HTTP receiver、MessagePack 解码、元数据持久化时
  - 当前直接相关：
    - `BE-021`
    - `BE-022`
- 测试工程师
  - 当需要把 gateway fixture、quarantine 测试、真实硬件联调验证落成回归或现场验证时
  - 当前直接相关：
    - `TE-020`
- 调度 / 用户
  - 当需要判断设备注册线是否从准备态升级到主线实现态时

## 7. 谁依赖你

当前依赖设备注册工程师口径的角色：

- 后端工程师
  - 依赖设备生命周期规则
  - 依赖未注册设备 quarantine 策略
  - 依赖删除后再上报的处理口径
- 测试工程师
  - 依赖设备注册侧预期行为来设计 fixture、回归和硬件联调判定
- 文档 / 调度侧
  - 依赖你维护设备注册、设备台账、网关准备态的统一口径

# 业务与调度总览（中文）

Last updated: 2026-03-22

这份文档有两个用途：

1. 给用户自己看：快速理解当前项目到底在做什么、主业务链是什么、当前调度规则是什么。
2. 给我自己看：当上下文变长、开新窗口、被多次打断后，用它快速恢复业务逻辑、调度逻辑和必须遵守的规则。

如果这份文档与用户最新明确指令冲突，以**最新用户指令**为准，然后尽快同步更新文档。

## 1. 当前项目是什么

- 项目名称：AIoT 智慧养老健康监测平台
- 前端：Vue 3 + TypeScript + Vite
- 后端：FastAPI
- 当前阶段：前端统一视觉逻辑 + 页面拆分 + 演示收口

当前最重要的事，不是继续堆新功能，而是：

1. 前端统一视觉逻辑
2. 登录/注册链拆成小页面
3. 报告 / 评估 / 告警主链稳定可讲
4. 任务板、文档、上报和代码保持一致

## 2. 当前主业务链

### 登录 / 注册链

当前固定为：

1. 登录页
2. 身份选择
3. 注册（只做账号）
4. 资料完善
5. 完成态 / 进入系统

说明：

- 注册阶段不做设备绑定
- 注册阶段不做老人/家属关系绑定
- 先进入系统，再做后续业务动作

### 绑定链

1. 老人注册
2. 家属注册
3. 在“成员与设备”里完成关系绑定
4. 家属刷新后获得对应老人访问能力

### 设备链

- 注册设备
- 绑定设备
- 解绑设备
- 换绑设备
- 删除设备
- 查看绑定历史

### 数据链

- 实时数据
- 趋势数据
- 智能评估
- 健康报告
- 异常到报警

### 报告链

报告必须走：

- `POST /api/v1/chat/report/device`

不再允许：

- `/chat/analyze`
- `/chat/analyze/device`

伪装成正式报告。

### 异常 / 报警链

- 正常
- 异常
- 持续异常
- 报警

### 网关链

当前状态：

- BLE HTTP 契约已定
- 实现未完全落地
- 未注册设备进入 quarantine

## 3. 当前产品与前端展示规则

### 当前视觉逻辑

- 登录页已经成为全站视觉基线
- 现在不再局限于“必须先有参考图才推进页面”
- 当前目标是：**稳定一个统一视觉逻辑，并向全站扩展**

### 当前视觉基线

- 绿色主基调
- 白色 / 淡绿色 / 淡蓝色层级
- 轻玻璃拟态卡片
- 医疗科技感但不过分炫技
- 中文优先

### 登录卡片尺寸基线

- 宽度：`600px`
- 最小高度：`360px`
- 圆角：`24px`
- 内边距：`36px`
- 输入框 / 按钮高度：`48px - 52px`

原则：

- 做成横向长方形面板
- 不做细长竖卡片

## 4. 对话模型与报告模型的新要求

当前真实情况：

- 普通对话和健康报告原先没有分成两套独立模型配置
- 它们曾共用同一个本地模型选择路由，只是 prompt / query 不同

用户最新决定：

- **普通对话模型** 和 **健康报告模型** 现在要正式分离
- 允许新增内部模型路由和内部模型配置
- 健康报告模型应该调用健康模型，生成规范式正式报告
- 前端必须把：
  - 对话入口 / 对话展示区
  - 报告入口 / 报告展示区
  分开放置，不再混在一起

实现原则：

- 可以新增内部路由与内部模型配置
- 不要轻易改公共 report schema
- 前端语义分离要先于或至少同步于模型路由分离

## 5. 当前调度逻辑

### 立刻启动

- 前端
- 后端
- 测试
- 离线模型
- 智能体
- 健康模型

### 跟进启动

- UI

### 条件启动

- 设备注册

### 当前调度原则

1. 先判断是不是主链阻塞
2. 再判断任务属于谁的主责任
3. 再判断是否需要并行
4. 最后才派发

## 6. 我必须遵守的规则

这是用户明确说过、我必须持续遵守的：

1. 不要忽视用户刚发的内容
2. 被打断后处理完插入事项，要继续原任务
3. 任务必须发布，不只分析
4. 上报必须严格审核
5. 没有完整上报，不派下一轮
6. 浏览器效果优先于终端输出
7. 环境不能混
8. 任务板、handoff、代码要尽量一致

## 7. 长期记忆 vs 可覆盖项

### 属于长期记忆的

- 前端是 Vue 3 + TypeScript
- Python / pytest 只走 `helth`
- 报告接口是 `/api/v1/chat/report/device`
- 浏览器效果优先
- 登录页是当前视觉基线
- 登录/注册链是 5 步
- 上报必须审核
- 没上报不派下一轮

### 可以被后续覆盖的

- 当前第一优先级任务是哪条
- UI 页面开放范围
- 是否继续扩全站视觉
- 当前背景图使用哪一张
- 是否要加粒子
- 是否立刻切数据库专项
- 是否提级 DR / AG / OM / HM

# 前端全站统一视觉重构协同方案

Last updated: 2026-03-22

## 目标

把当前前端从“页面风格各自生长、结构混杂、说明文字冗长”的状态，重构成：

- 有统一视觉逻辑
- 有统一页面层级
- 有统一按钮 / 卡片 / 表单 / 图表语言
- 登录页作为视觉基线
- 业务页沿用同一套基线扩展

这不是简单修样式，而是：

- 统一设计系统
- 统一页面职责
- 统一展示逻辑
- 保持业务主链可演示

## 当前统一视觉基线

- 绿色主基调
- 白色 / 淡绿色 / 淡蓝色层级
- 轻玻璃拟态卡片
- 安静、医疗、科技、可信赖
- 中文优先，不堆内部开发说明

## 登录页作为基线

登录页当前已经承担的角色：

- 定义卡片比例
- 定义按钮语言
- 定义背景层级
- 定义表单输入框样式
- 定义反馈状态与标签体系

### 登录卡片推荐规格

- 宽度：`600px`
- 最小高度：`360px`
- 内边距：`36px`
- 圆角：`24px`
- 输入框高度：`48px - 52px`
- 按钮高度：`48px - 52px`

原则：

- 做成横向长方形面板
- 不要做成细长竖卡片

## 全站页面统一扩展顺序

### 第一批

- 登录页
- 登录/注册链路
- 健康报告页

### 第二批

- 家属首页
- 社区总览
- 成员与设备

### 第三批

- 告警页
- 趋势页
- 设备接入 / 数据模式

## 页面命名规范

- 登录
- 身份选择
- 创建账号
- 完善资料
- 注册完成
- 首页
- 健康趋势
- 健康报告
- 异常提醒
- 总览
- 成员与设备
- 告警中心
- 设备接入
- 数据模式
- AI 健康助手

不再使用：

- “关系台账”
- 模糊的英文内部术语
- 开发说明式标题

## 页面结构原则

### 登录/注册链

必须拆成小页面：

1. 登录页
2. 身份选择页
3. 注册账号页
4. 资料完善页
5. 完成态页

### 业务页

每页只承担一个核心任务：

- 首页看总况
- 趋势页看变化
- 报告页看结论
- 告警页看风险与处理
- 成员与设备页看绑定与管理

不再做：

- 一个页面里同时堆登录、注册、设备、关系、报告、调试信息

## 报告与对话展示规则

当前新增一条明确规则：

- **普通 AI 对话** 与 **正式健康报告** 必须在前端分开展示

即使后端当前仍可能走同一路由选择逻辑，前端也要明确分开：

- AI 对话区
- 健康报告区

不允许继续把两者放在同一个展示容器里造成语义混乱。

## 组件统一清单

建议逐步抽离这些公共组件：

- 通用页面壳
- 页头
- 玻璃卡片
- 普通信息卡
- 指标卡
- 状态标签
- 主按钮 / 次按钮 / 幽灵按钮
- 输入框
- 搜索框
- 表格工具条
- 空态
- 加载态
- 错误态
- 图表容器

## 角色协同边界

### UI 设计师

- 负责统一视觉逻辑与结构规范
- 不改业务逻辑

### 前端工程师

- 负责按统一逻辑落地
- 负责组件抽离与页面拆分

### 后端工程师

- 负责契约稳定与小阻塞修复
- 不负责视觉决定

### 测试工程师

- 负责验证重构后主链是否仍成立

## 当前最重要的结论

现在已经不再是“只有参考图页面才能推进”的阶段。  
当前阶段应该：

- 稳定登录页这套视觉逻辑
- 抽成可复用的设计语言
- 向全站扩展

但扩展时必须：

- 沿用统一基线
- 不要每页单独发明一套风格
- 不要破坏真实业务主链

# 登录页动态背景实现规范

Last updated: 2026-03-22

## 目标

基于用户提供的登录参考图和纯净背景图，实现一个“有生命感、轻量、可演示”的登录页动态背景。

目标不是做复杂特效，而是让登录页具备：

- 医疗科技感
- 健康监测主题感
- 视觉中心明确
- 中文标题和表单清晰可读
- 桌面端有动效，移动端可降级

## 总体方案

优先采用：

- 静态背景图
- SVG 心电图
- 渐变扫光
- 少量粒子点缀（可选）

不推荐第一版直接做：

- 重型 Canvas 动画
- 高频复杂波形
- 大量全屏粒子

## 分层结构

### 第一层：静态底图

来源：

- 用户提供的纯净背景图

作用：

- 提供蓝绿医疗科技氛围
- 保持页面干净
- 让登录卡片有柔和承托

要求：

- 全屏铺底
- 不能压过登录卡片
- 左右视觉元素尽量完整保留

### 第二层：动态心电图层

作用：

- 提供“健康监测系统”的主题感
- 在不增加信息噪音的情况下，增加生命感

要求：

- 一条 SVG 心电图主线
- 透明度较低
- 颜色以蓝青 / 青绿为主
- 与背景图风格一致

### 第三层：登录内容层

包含：

- 系统标题
- 登录卡片
- 注册入口
- 辅助说明

要求：

- 永远是视觉中心
- 动效不能穿过输入框文字区域

## 登录卡片设计规范

## 卡片定位

登录卡片必须是页面唯一明确的交互中心。

要求：

- 第一眼先看到卡片，再看到背景动效
- 卡片不是漂浮小窗，而是清晰的主操作容器
- 卡片必须承担：
  - 登录
  - 注册入口
  - 当前流程说明

## 卡片结构

建议固定为 5 段：

1. 标题区
2. 输入区
3. 主按钮区
4. 辅助链接区
5. 注册入口区

### 1. 标题区

包含：

- 主标题：`登录账户`
- 辅助文案：一句话说明这是进入系统的入口

要求：

- 不要堆太多说明
- 标题和副标题层级清楚
- 全中文

### 2. 输入区

包含：

- 账号输入框
- 密码输入框

要求：

- 输入框高度统一
- 左侧图标可保留，但必须弱化
- placeholder 全中文
- 错误态边框和错误文案明确

### 3. 主按钮区

包含：

- `登录` 主按钮

要求：

- 是整张卡片里最明确的 CTA
- 颜色用主品牌色
- hover / active / disabled 状态完整

### 4. 辅助链接区

包含：

- `忘记密码？` 或 `需要帮助？`

要求：

- 不能和主按钮抢层级
- 位置清晰但弱于主按钮

### 5. 注册入口区

包含：

- `还没有账号？立即注册`

要求：

- 必须比“普通页脚链接”更明显
- 这是主链路的一部分，不是次要信息
- 点击后进入：
  - 身份选择
  - 注册（只做账号）
  - 资料完善
  - 进入系统

## 卡片视觉要求

### 背景

- 白色或接近白色
- 允许轻玻璃感，但不能发灰

### 圆角

- 建议 `20px - 28px`

### 阴影

- 柔和、大范围、低对比
- 不要厚重投影

### 宽度

- 桌面端建议 `420px - 520px`
- 不宜过窄
- 不宜超过页面中心视觉占比

### 留白

- 卡片内部上下左右留白要充分
- 不要做成“表单挤在一起”

## 文案要求

必须全中文产品化表达：

- `登录账户`
- `请输入账号`
- `请输入密码`
- `登录`
- `忘记密码？`
- `还没有账号？立即注册`

不要在登录卡片里出现：

- 过多英文标签
- 技术接口名
- 调试说明
- “mock / analyze / report mode” 这类内部词

## 响应式要求

### 桌面端

- 卡片居中
- 标题在卡片上方或卡片内上部

### 平板端

- 卡片保持中心
- 适度缩宽

### 手机端

- 卡片宽度接近全宽
- 阴影减弱
- 背景动效减弱
- 仍保持注册入口明显

## 动效要求

### 方案 A：SVG 心电图 + 渐变扫光

这是默认方案。

#### 动效组成

1. 常态心电线
2. 沿线移动的高亮渐变
3. 峰值区域轻微 glow

#### 节奏建议

- 一个完整循环：`3.2s - 4.2s`
- 不要快于 `2.8s`
- 不要慢于 `5.5s`

#### 强度建议

- 常态线透明度：`0.18 - 0.28`
- 扫光高亮透明度：`0.55 - 0.75`
- glow blur：`6px - 12px`

#### 视觉要求

- 扫光是“轻扫过去”，不是整条闪烁
- 高光不能太刺眼
- 动画节奏要平稳，不要像故障抖动

### 方案 B：少量粒子点缀

只有在方案 A 仍然显得过静时，才补这一层。

#### 粒子形式

- 少量圆点
- 沿心电线附近缓慢漂移
- 或在峰值处短暂发光

#### 数量限制

- 桌面端：同时可见粒子不超过 `8`
- 移动端：同时可见粒子不超过 `3`

#### 禁止项

- 满屏粒子雨
- 星空粒子
- 长拖尾
- 游戏感发光特效

## 响应式要求

### 桌面端

- 保留完整背景图
- 保留心电图和扫光效果
- 登录卡片居中

### 平板端

- 保留背景图
- 心电图透明度降低
- 登录卡片缩小

### 手机端

- 保留静态背景图
- 可只保留轻量扫光
- 粒子默认关闭
- 如性能不稳，直接去掉 ECG 动效，只保留静态氛围

## 前端实现优先级

1. 背景图接入
2. SVG 心电图层
3. 渐变扫光动画
4. glow 细节
5. 粒子（可选）

## 技术建议

- 背景图：CSS 背景或独立绝对定位层
- 心电图：SVG path
- 扫光：CSS animation + SVG gradient / mask
- 粒子：轻量 CSS keyframes 或极少量 JS 驱动

## 验收标准

完成后至少满足：

1. 登录页第一眼有健康监测系统气质
2. 登录卡片仍然是视觉焦点
3. 中文标题和输入框可读性不受影响
4. 动效不花、不吵、不廉价
5. 手机端不掉帧
6. 即使不加粒子，效果也成立

## 协同分工

### UI Designer

- 负责定义：
  - ECG 位置
  - 扫光节奏
  - glow 强弱
  - 粒子是否需要
  - 桌面 / 移动端差异

### Frontend Engineer

- 负责实现：
  - 背景图接入
  - SVG 心电图
  - 渐变扫光
  - 移动端降级
  - 必要时补极少量粒子

## 备注

顺序必须是：

先把心电图 + 渐变扫光做对  
再决定要不要加粒子

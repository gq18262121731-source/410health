# 摄像头视频流问题解决方案

## 🔍 问题诊断

### 原始问题
前端显示摄像头在线，但视频区域全黑，无法显示画面。

### 根本原因
**WebSocket端口配置错误**：
- 前端运行在 `http://localhost:9000`
- 后端运行在 `http://localhost:8000`
- 前端自动推导WebSocket地址为 `ws://localhost:9000/ws/camera`
- 但WebSocket服务器实际在 `ws://localhost:8000/ws/camera`
- **结果**：前端连接到错误的端口，导致WebSocket握手超时

### 错误表现
```
❌ WebSocket connection to 'ws://localhost:9000/ws/alarms' failed: WebSocket opening handshake timed out
❌ Failed to load resource: net::ERR_NETWORK_CHANGED
❌ Failed to load resource: net::ERR_METHOD_CHANGED
```

## ✅ 解决方案

### 1. OpenCV异步问题（已解决）
**问题**：主后端在异步环境中无法直接访问本地摄像头
- OpenCV在FastAPI的异步上下文中抛出C++异常
- 所有后端（DSHOW, MSMF, ANY）都失败

**解决**：创建独立的摄像头服务进程
- 文件：`camera_service_standalone.py`
- 端口：8001
- 功能：独立进程运行，避免异步冲突
- 提供：`/snapshot` 和 `/stream.mjpg` 端点

### 2. 主后端集成（已解决）
**配置**：
```env
CAMERA_SOURCE_MODE=local
CAMERA_LOCAL_INDEX=0
CAMERA_LOCAL_HTTP_URL=http://localhost:8001
```

**实现**：
- 主后端优先使用HTTP快照服务（`camera_local_http_url`）
- 回退到直接访问摄像头（如果HTTP服务不可用）
- 配置文件：`backend/config.py` 添加 `camera_local_http_url` 字段
- 服务文件：`backend/services/camera_service.py` 修改 `capture_local_jpeg()` 方法

### 3. 前端WebSocket配置（核心解决方案）
**问题**：前端WebSocket地址推导错误

**解决**：创建前端环境配置文件
```bash
# frontend/vue-dashboard/.env
VITE_API_BASE=http://localhost:8000/api/v1
VITE_WS_BASE=ws://localhost:8000
```

**原理**：
- Vite会读取`.env`文件中的环境变量
- `VITE_WS_BASE` 明确指定WebSocket服务器地址
- 前端代码会优先使用环境变量，而不是自动推导

## 🎯 技术要点

### 1. 为什么需要独立摄像头服务？
- **异步冲突**：OpenCV在FastAPI的异步环境中不稳定
- **资源隔离**：独立进程避免主后端的异步操作影响摄像头访问
- **简单可靠**：独立服务使用同步代码，更稳定

### 2. 为什么WebSocket配置如此重要？
- **跨域问题**：前端和后端在不同端口
- **自动推导失败**：Vite开发服务器的origin与后端不同
- **明确配置**：环境变量确保正确的连接地址

### 3. 视频流架构
```
本地摄像头 (USB)
    ↓
独立摄像头服务 (8001)
    ↓ HTTP快照
主后端 (8000)
    ↓ WebSocket
前端 (9000)
    ↓
浏览器显示
```

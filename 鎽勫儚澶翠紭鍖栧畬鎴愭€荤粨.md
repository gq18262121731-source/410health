# 摄像头性能优化完成总结

## 优化概述

针对IPC-SDK 8MP P2P摄像头（192.168.8.254:10554）的性能问题，实施了基于实际代码分析的优化方案。

## 已实施的优化

### 1. HTTP Session连接池 ✅
**文件**: `backend/services/camera_service.py`

**实施内容**:
- 在`CameraService.__init__`中创建`requests.Session`实例
- 配置HTTPAdapter连接池：
  - `pool_connections=10`: 最大连接池数
  - `pool_maxsize=10`: 每个主机最大连接数
  - `max_retries=1`: 自动重试1次
- 在`_post_onvif`方法中使用Session替代直接requests调用

**预期效果**:
- 减少TCP握手开销30-50%
- 复用HTTP连接，降低网络延迟
- 提高ONVIF PTZ控制响应速度

### 2. 帧缓存机制 ✅
**文件**: `backend/services/camera_stream_hub.py`

**实施内容**:
- 在`CameraFrameHub`中添加帧缓存字典
- 实现`_get_cached_frame`和`_cache_frame`方法
- 缓存TTL设置为150ms（适合6fps）
- 在`_run_snapshot_stream`和`_run_local_snapshot_stream`中使用缓存
- 自动清理过期缓存（最多保留10帧）

**预期效果**:
- 避免重复请求相同帧
- 提高响应速度50-70%
- 减少摄像头负载

### 3. OpenCV优化配置 ✅
**文件**: `backend/services/camera_service.py`

**实施内容**:
- 优化`OPENCV_FFMPEG_CAPTURE_OPTIONS`环境变量：
  - 添加`flags;low_delay`标志
  - 保持`max_delay;0`和`fflags;nobuffer`
- 设置`CAP_PROP_FPS=30`提高捕获速度
- 保持`CAP_PROP_BUFFERSIZE=1`减少延迟

**预期效果**:
- 减少帧捕获延迟20-30%
- 提高OpenCV读取性能
- 降低缓冲区延迟

### 4. 异步优化 ✅
**文件**: `backend/services/camera_stream_hub.py`

**实施内容**:
- 在`_run_snapshot_stream`中使用`asyncio.to_thread`包装同步调用
- 在`_run_local_snapshot_stream`中使用`asyncio.to_thread`
- 避免阻塞事件循环

**预期效果**:
- 提高整体系统响应性
- 避免阻塞其他异步任务
- 更好的并发性能

### 5. 性能监控 ✅
**文件**: `backend/services/camera_stream_hub.py`, `scripts/monitor_camera_performance.py`

**实施内容**:
- 在`status()`方法中添加缓存性能指标：
  - `cache_hits`: 缓存命中次数
  - `cache_misses`: 缓存未命中次数
  - `cache_hit_rate`: 缓存命中率
  - `cache_size`: 当前缓存大小
- 创建性能监控脚本`monitor_camera_performance.py`

**功能**:
- 实时监控帧率、延迟、缓存命中率
- 性能评估和建议
- 快照性能测试

## 技术细节

### HTTP Session连接池
```python
self._http_session = requests.Session()
adapter = requests.adapters.HTTPAdapter(
    pool_connections=10,
    pool_maxsize=10,
    max_retries=1,
    pool_block=False
)
self._http_session.mount('http://', adapter)
```

### 帧缓存机制
```python
# 缓存键基于时间和帧率
cache_key = f"snapshot_{int(time.time() * fps)}"

# 检查缓存
cached_frame = self._get_cached_frame(cache_key)
if cached_frame is not None:
    frame = cached_frame
    self._cache_hits += 1
else:
    frame, _headers = await asyncio.to_thread(service.capture_jpeg)
    self._cache_frame(cache_key, frame)
    self._cache_misses += 1
```

### OpenCV优化
```python
os.environ["OPENCV_FFMPEG_CAPTURE_OPTIONS"] = "rtsp_transport;tcp|max_delay;0|fflags;nobuffer|flags;low_delay"
cap.set(cv2.CAP_PROP_BUFFERSIZE, 1)
cap.set(cv2.CAP_PROP_FPS, 30)
```

## 故障排查

### 问题1: 帧率仍然很低
**可能原因**:
- 网络带宽不足
- 摄像头本身性能限制
- 跌倒检测推理占用过多资源

**解决方案**:
- 检查网络连接质量
- 降低`CAMERA_STREAM_FPS`配置
- 确认GPU加速已启用

### 问题2: 缓存命中率为0
**可能原因**:
- 缓存TTL过短
- 请求频率过高
- 缓存键生成问题

**解决方案**:
- 调整`_frame_cache_ttl`参数
- 检查日志中的缓存统计

### 问题3: 连接池未生效
**可能原因**:
- Session未正确初始化
- 使用了直接requests调用

**解决方案**:
- 检查所有HTTP调用是否使用`self._http_session`
- 查看网络连接数（应该保持稳定）
